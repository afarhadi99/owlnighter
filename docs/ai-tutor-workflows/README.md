# AI Tutor API workflow: quiz generation

`quiz-generation-workflow.json` is generated by `build.mjs` â€” **regenerate it
with `node docs/ai-tutor-workflows/build.mjs`** whenever the quiz framing text,
the named-variable spec, or `apps/api/src/services/quiz.ts`'s `GeneratedQuiz`
schema changes; never hand-edit the JSON file directly.

## What it is

- This is a **dedicated, templated** quiz workflow â€” not the old generic
  `{{system}}\n\n{{user}}` passthrough. The framing (system) instructions are
  **baked into `template`**, followed by a labeled layout that interpolates six
  discrete named inputs: `stepTitle`, `chapterHint`, `pageRange`, `quizMode`,
  `questionCount`, and `readerContext`. `inputs` declares exactly those six
  names.
- owlnighter's `AiTutorApiAdapter` (`packages/ts/ai/src/aiTutorApi.ts`) sends a
  flat request body. For quiz generation it sends the **named variable map**
  (`opts.variables`, built by `quizVariables()` in `quiz.ts`) whose keys match
  the `inputs` names above. Tasks that have **not** been migrated to their own
  templated workflow (e.g. `rewrite`) still work through the adapter's generic
  `{ system, user }` **fallback** body â€” the adapter picks the variable map when
  a task supplies one and falls back to `{ system, user }` otherwise.
- The framing is **single-sourced from `build.mjs`**: the same run that writes
  this JSON also emits `apps/api/src/services/quiz-prompt.generated.ts`, which
  exports `QUIZ_SYSTEM_FRAMING`. `quiz.ts` imports that constant and passes it as
  `system` for the Gemini/Groq/OpenRouter path (those providers compose their
  prompt from `system` + `user` and ignore `variables`). Because both the baked
  template framing and the app-side constant come from the one generator, the
  two cannot silently drift.
- `modelSettings.enableWebSearch: true` â€” quiz accuracy benefits from live
  search grounding.
- `modelSettings.structuredOutputSchema` is the real JSON Schema for
  owlnighter's `GeneratedQuiz` shape, generated from the same object every time
  `build.mjs` runs â€” so it can never silently drift from
  `apps/api/src/services/quiz.ts`'s actual Zod schema.
- `model: "gemini-3.5-flash"` is *this platform's own* model catalog entry
  (confirmed, by reading `manage-prompt`'s `data/workflow.ts` capability map,
  to support both `webSearch` and `structuredOutput` and not be deprecated) â€”
  it is unrelated to owlnighter's own `GEMINI_MODEL` env var / Gemini adapter;
  don't confuse the two even though the name looks similar.

## How to use it

1. In your AI Tutor API console, import this file (drag-and-drop the
   `.json`, or `PUT https://aitutor-api.vercel.app/api/workflows/import` with
   the file's bytes as the request body and your account's Bearer key).
2. Note the `shortId` (`wf_...`) the platform assigns on import â€” this file's
   own `name` field is overwritten by the import route regardless of what's
   in the file, so don't expect the name you see in the JSON to survive.
3. In owlnighter's admin panel, go to **AI Providers â†’ AI Tutor API** and
   paste that id into **Workflow ID â€” quiz generation**.
4. Set an AI Tutor API key in the same card, and optionally reassign quiz
   generation to `ai_tutor_api` under **Default provider & task overrides**.

Importing is a manual, explicit step by design â€” the Bearer key belongs to a
real, possibly-billable account, so owlnighter never calls
`PUT /api/workflows/import` automatically on anyone's behalf.
